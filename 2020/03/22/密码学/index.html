1 密码学基础

加密三要素

  1. 明文： 你要发送的数据

  2. 加密算法 ： 一身那么样的方式进行加密（des 3des aes）

  3. 密钥： 双方规定的钥匙 会根据算法的不同，长度不同

解密三要素：

  1. 密文
  2. 解密算法： 可能和加密算法相同，也可能不同（亦或加密=》 算法相同，凯撒密码=》》算法不同）
  3. 秘钥 一定与加密的密钥相同

凯撒密码

hello world -》 加密_—> khoor zrug—>>解密–》hello world

加密过程：

  1. 明文： hello world
  2. 算法 ： 向右移动
  3. 密钥： 3

解密过程：

  1. 密文： khoor zrug
  2. 算法： 向左移动（与加密算法不同）
  3. 密钥 ：3

2 加密方式

对称加密 vs 非对称加密

对称加密：

  1. 密钥的数量 1个

  2. 特点：

    1. 加密效率高
    2. 双方使用的密钥相同

  3. 安全性

    1 不安全 （相对于非对称加密）

  4. 使用情况

    1. 主流的加密方式

非对称加密：

  1. 密钥数量：2个

    1. 公钥

      • 任何人都可以持有

      • 一般用于加密作用

    2. 私钥

      • 只有自己持有
      • 一半呢不用于加密，而是用于签名
      • 签名的数据可以证明是私钥的持有人发送数据
      • 私钥签名的数据，私钥的持有人无法否认自己发送过这个消息

  2. 特点

    1. 公钥加密的只有自己的私钥能解开
    2. 加密解密效率低一般不做大量的数据使用

  3. 安全性

    1. 安全性高

  4. 使用情况

    1. 一般配合对称加密使用
    2. 建立连接之初，先使用非对称加密协商好对称加密算法和密钥
    3. 然后使用对称加密进行后续的解密

三、 秘密信息安全常识与威胁

密码信息安全常识

  • 不要使用保密的密码算法

  • 使用低强度的密码比不进行任何加密更加危险

  • 任何密码都有被破解的一天（分对称加密算法由英国安全局1973年发明，后RSA发布于1977年）

  • 密码知识信息安全的因部分

五 常用的加密算法

1 DES

Data Encryption Standard

特点：

  1. 不安全，不建议使用
  2. 密钥：8字节64bit（其实只使用了56位，每7个bit就会设置一个校验位 ）
  3. 加密的时候会对明文进行分组，分组长度是8bytes 加密之后得到的密文也是8byte

2 3DES

Triple Data Encryption

  1. 加密过程：

​ 加密-》解密-》加密

特点：

​ A 中间使用解密的原因是为了兼容以前的DES

​ B 解密过程是以解密的方式进行加密 整体还是三次加密

​ C 密钥： 8byte * 3 = 24bytes => 24 * 8 = 192bit

​ D 数据分组从航都： 与DES相同，8Byte

​ E: 复杂的加密算法 效率低（过渡的加密算法）

​ F 少用

  1. 解密过程

解密 -》加密-》解密

  1. 3个密钥

    • 如果密钥1 和密钥1相同，或者密钥2与密钥三相同 就相当于DES 与前面进行兼容

    • 如果密钥1密钥3相同，相当于有两个密钥 专业名字：3des-EDE3

    • 如果三个密钥都不相同的，专业名字 3des-EDE3

3 AES

Advacnced Encryption Standard

替代传统的DES算法

AES规格 128 192 256

特点：

  1. 密钥的长度： 可以选择的 16bytes 24bytes 32bytes传统的des是8bytes， 3des 24bytes
  2. 分组长大衣： 16bytes （des和3des： 8bytes）
  3. 加密解密效率高（推荐使用）

为什么要进行分组

被加密的数据可能很大

需要对数据进行迭代的加密，所以对数据进行分组

明文分组与密文分组

  • 密文分组的长度与密文分组移植

一共有五种分组模式

ECB Ekectronic Coding Book 电子密码本模式 ===》淘汰

CBC Cypher Block Chaining 密文分组连接模式===》》 常用的

CFB Cyper Feedback 图文反馈模式，建议使用CTR

OFB Output FeedBack 输出反馈模式，建议使用CTR

CTR Count 计数器模式

加密算法与语言是没有关系的

七、具体的模式

1 ECB模式 Electronic Coding Block 电子

一个被攻克了之后 后面就全被破解了

特点：

  • 加密效率高但是不安全 加密不彻底
  • 需要对数据分组后进行填充
  • 每一个分组独立的进行加密解密
  • 不用 使用，go语言没有支持这种模式
  • 只要一个分组被破解，那么后面所有的分组都被破解
  • 分组的长度由加密算法决定

​ des: 8byte aes:16byte
