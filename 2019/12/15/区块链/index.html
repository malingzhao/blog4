区块链产生：

  2. 不可篡改文件记录：加盖时间戳到文件内容中，证实数据是真实存在的，保证安全

  3. 数字现金：

  4. 发展历史：

    1. 09 年比特币诞生1.0
    2. 14年以太坊诞生2.0
    3. 18年中期的时候，EOS主网上线 3.0
    4. Fabric： 联盟链

​ 最早诞生最具有代表意义的

  1. 区块链的场景概念

    1. 数字货币：可以与现金流等价交换 拥有现金流的属性和功能
    2. ICO： 众筹
    3. 电子钱包: 在电商的购物中的常用的支付工具

  2. 什么是区块链（定义)

  3. 区块链是一种集成了点对点协议、现代密码学、共识算法、分布式数据库新型的应用模型

  4. 区块链的特点

    1. 可追溯
    2. 不可篡改
    3. 去中心化
    4. 完整备份： 区块链具有完整的分布式存储特性
    5. 历史记录： 被存储的数据拥有完整的历史记录可以快速查看复原
    6. 交易广播：一次交易分发给网络中的其他节点， 同步进行接收

  5. 区块链加密货币的特点：

    1. 独立性： 所有的货币是独立存在的
    2. 唯一性： 地址，交易都具有不重复的唯一性
    3. 匿名性：账户信息和个人信息没有关联，整个交易的过程全局加密
    4. 不可伪造 ：

  6. 区块链核心技术

    1. 点对点传输协议：在网络中数据的流通方式

    2. 现代密码学： 在 区块链中的应用：公钥、私钥的签名

    3. 共识算法：数据的一致性

    4. 分布式数据存储：开发质量

      Java语言、C++语言、MapReudce、Hadoop用到Go的方面 Go语言的web开发 有分布式的开发经验

      实现去中心化的重要技术依据

  1. 区块链的核心概念

    1. 区块链

      1. 本质： 一个分布式账本通过你共识算法来决定谁能抢到当前的几张券，区块链以区块为单位，以区块产生的时间顺序去进行连接

      2. 区块：

        1. 概念： 区块链的基本组成单位
        2. 区块头：
          1. 时间戳
          2. 当前的区块Hash
          3. 父区块Hash
          4. 随机数
          5. Merkle树
          6. 区块的号码
        3. 区块体： 交易数据

      3. 分布式数据库：区块链中的区块数据存储在每一个节点中，所有的节点组成一个分布式数据库，任何一个或者多个节点退出都不会影响其他节点，因为每个节点都保存了完整的数据

      4. 节点

        1. 可以理解成一个运行区块软件的计算机
        2. 分类
          1. 全节点：保存了完整的区块链副本，安全性极高，效率不高
          2. 轻节点： 不保存所有的区块，需要全节点进行验证 ，效率更高，安全性不如安全节点

      5. 挖矿节点：

        1. 带有挖矿功能的全节点 专门处理交易验证
        2. 穷举随机数算法生成hash，与目标哈希进行比较

      6. 分叉：

        1. 升级分叉 ： ETC和ETH 升级分差 bug 解决掉 两个功能实现分差
        2. 挖矿分叉：
          1. 升级分叉： 两个或者多个旷工同时完成了工作量证明 产生两个新的分叉
            1. 旷工遵从不同的机制（规则）导致分叉
            2. 硬分叉 ：如果区块链共识规则改变之后不允许前向兼容，纠结点可以兼容新节点产生的区块
            3. 软分叉：如果区块链共识规则改变之后允许前向兼容，旧结点没有办法认可新节点产生分叉
          2. 解决方案：不同的旷工跟随者不同的区块，但是不同的链的算力是不一样的，旷工的数量不一样，链的增长速度就不不相同 ，最终出现一条链更好，这条链就变成主链

      7. 51%攻击 一旦到达了51%之后就会得不偿失

      8. 交易

        1. 概念： 一笔资产在参与者之间的转移
        2. 内容
          1. 金额
          2. 发送者
          3. 接收者
          4. 交易ID（hash）

      9. 双花：

        1. 基本概念： 复用数字货币的数字特性可以完成两次或者多次的支付
        2. 传统的虚拟货币之所以能够避免双花是因为有可依赖的第三方机构避免双花
        3. 区块中需要达成分布式节点之间的相互校验与共识机制来避免双花，同时完成价值的转移

      10. UTXO(Unspent transaction output )交易模式

        1. 是比特币的特有的交易模式，比特币交易过程中的基本方式，主要为了避免双华

      11. 哈希

        1. 将任意的原始的数据交易记录通过制定的哈希函数 编码为特定长度的字符串

        2. 在区块链中的应用：生成地质 交易验证

        3. 特点

          1. 不可逆
          2. 随机性
          3. 时间正相关：输入的源数据越长 ，hash处理的时间会更长

        4. 加密算法

          1. 对称加密： 加密与解密 相同的密钥
          2. 非对称加密
            1. 采用公钥和私钥进行加密
            2. 无法用公钥反推私钥

        5. 数字签名

        6. Merkle树

          1. Merkle树可以是二叉树 也可以是多叉树，具有树的所有的特点
          2. 在区块链中作用：
            1. 快速校验
            2. 归纳交易数据的完整性
            3. Merkle树可以极大的提高查询效率，区块头只需要保存一个Merkle树根的hash
            4. Merkle树支持SPV

        7. P2P

          1. 通过对等网络来分配工作任务 ，分布式的应用架构
          2. 迅雷采用的就是P2P
          3. 由于在P2P中所有的网络节点是对等的 不存在一个中心化节点，也不存在所谓的层级结构，所以每个节点都要承担验证区块数据等功能
9. 区块链的分类

   1. 公有链： 真正意义上的分布式区块链，任何一个节点都可以随时加入/退出网络中，

   2. 私有链： 部分中心化的区块链，具有一定的分布式特点但是有一个中心接地那可以指定参与者 

   3. 联盟链： 美国的一个区块链的联盟，

      部分去中心化的区块链具有权限控制的功能

      代表： Fabric

10 . 区块链的架构特点

  1. 去中心化：基于分布式系统 没有中心机构存在

  2. 可靠数据库： 分布式存储、参与系统的节点越多

    数据的安全性就越高

  3. 开源可编程 ： 区块链 提供： 灵活的脚本系统甚至于完善的开发平台，支持用户创建更高级的应用

  4. 集体维护： 区块链中的数据，整个系统所有记账功能的节点进行维护

  5. 安全可信： 通过现代密码学实现

  6. 准匿名性 ： 采用身份信息无关的哈希作为吧哈希地址与交易ID

  1. 典型应用分析

    1. 比特币： 2100万

      1. 特点

        1. 问题2100w 永不增发
        2. 挖矿奖励，逐年减半

      2. 架构

        1. 钱包： 保存用户私钥，管理用户约 ，提供比特币交易（支付、转账）

        2. 钱包分类

          1. 决定性钱包：所有的私钥由耦合私钥种子通过hash算法生成
            1. 普通的决定性钱包：由私钥种子一次性生成所有私钥
            2. 层级决定性钱包： 由私钥种子生成父私钥 ，父私钥生成子私钥
          2. 非决定性钱包，直接保存私钥，私钥直接放在DB上面
            1. 展示方式的分类
              1. 钱包
                1. 厚钱包： 下载整条链可以完整交易安全性高 验证成本高
                2. 薄钱包：不会下载整条区块，采用部分存储+节点验证的方式
                3. 离线钱包： USB设备，纸钱包
                  ，可以有效防范网络攻击
              2. Http/Json RPC API
                1. 比特币提供的接口 ，可以使得外部通过该接口访问或者控制比特币节点
              3. 命令行接口
                1. 通过命令行的方式实现类似钱包的功能
              4. 浏览器
                1. 访问区块链的区块数据信息

        3. 节点后台 ： 负责参数比特币的通信，区块链维护，验证交易。。。。

        4. 比特币地址：

          1. 基本概念 hash生成
          2. 生成过程
            1. 随机数生成私钥
            2. 采用Secp256k1椭圆加密算法生成公钥
          3. 生成地址
            1. 以公钥作为输入，进行SHA256 在进行RIPEMD160 最后通过base258编码生成比特币地址
          4. 比特币区块校验：确保确实完成了工作量证明
            1. 校验内容： 格式 难度 时间戳 大小 交易
          5. 比特币交易
            1. 交易结构
              1. 输入 （Input）
              2. 输出（Output）
              3. 交易的ID（HASH）
            2. 注意
              1. 每一笔交易的输入来自前面交易的输出
            3. UTXO数据库 ：专门用于存储当前比特币未被花费的
            4. 比特币的区块校验L确保完成了工作量证明

        5. 以太坊

          1. 以太坊的基本概念： 一个用于开发去中心化DAPP的分布式平台 （EVM虚拟机）
          2. 智能合约： 一个拥有自我校验与自我执行的协议
          3. 以太坊为什么可以被称为区块链2.0： 以太坊提供了非常方便的应用开发平台 对底层做了完善的封装 让开发者只专注上层应用
          4. 共识： POW（ETHASH）

        6. EOS

          1. EOS的基本概念：
            1. 一个用于开发去中心化和DAPP的分布式平台
            2. 与以太坊的比：
              1. TPS有了极大的提升 号称能够达到百万级的TPS处理
              2. 以太坊本身是一条公链，其中每个DAPP会消耗整个链上的资源，它是一个区块链的基础架构，开发者可以自由的在EOS上创建公链
              3. EOS不在是一条单纯的公链 它是一个区块链的基础架构，开发者可以自由的在EoS上创建公链，链与链之间不会影响彼此的资源的使用，因此不会出现单个应用占用的资源太多而使得整个网络的拥堵
            3. 共识算法： DPOS
            4. EOS上的智能合约不需要手续费

        7. Fabric （联盟链）

          1. 联盟链代表 ： 与公链的最大的区别在于不同的虚拟货币

          2. 目标： 实现一个通过权限管理的区块链的底层架构

          3. 架构分类

            1. 身份服务 Fabric需要身份识别能力，加入/退出Fabric需要权限认证
            2. 策略服务：提供访问控制，授权等一些列功能
            3. 区块链服务： 提供分布式账本的基础能力以及数据传输、共识打成的底层功能
            4. 智能合约：验证节点运行的分布式程序用于自动执行特定的业务规则
